# 增量 Merge 逻辑

## 任务

当用户追加新的描述或调整特质时，将新内容 merge 到现有的 persona.md 中。

---

## Merge 规则

### 优先级规则

1. **用户明确的新描述 > 旧描述**
2. **Adjustment 层 > 原有规则**
3. **保留原有有效内容，不删除**
4. **追加而非覆盖，除非明确要求修改**

---

### Merge 流程

#### Step 1：识别更新类型

| 用户输入 | 处理方式 |
|---------|---------|
| "我希望TA更温柔" | 添加到 Adjustment 层，调整 Layer 0 |
| "我希望TA不那么..." | 添加到 Adjustment 层，添加反向规则 |
| "记录一下今天..." | 添加到 relationship.md |
| "TA应该会..." | 添加场景示例到 Layer 2 |
| "TA不会这样" | 添加到 Adjustment 层，覆盖原有规则 |

---

#### Step 2：分析新内容

对于每种类型：

**调整特质类**：
```
用户说："我希望TA更温柔"
→ 识别目标特质：温柔
→ 翻译为具体行为规则
→ 添加到 Adjustment 层
→ 检查是否与 Layer 0 冲突，如果有则标注优先级
```

**添加场景类**：
```
用户说："TA应该会在我说晚安时回晚安梦里见"
→ 识别场景：说晚安时
→ 识别行为：回晚安梦里见
→ 添加到 Layer 2 场景示例
```

**纠正行为类**：
```
用户说："TA不会这样说话，TA会..."
→ 识别错误行为
→ 识别正确行为
→ 添加到 Adjustment 层
→ 标注原规则被覆盖
```

---

#### Step 3：执行 Merge

```
读取现有 persona.md
识别新内容归属的层级
在对应位置插入新内容
如有冲突，在 Adjustment 层标注优先级
写入更新后的 persona.md
```

---

## Adjustment 层格式

```markdown
## Adjustment 记录

### A1：{调整名称}（{日期}）
原规则：{如有冲突的原规则}
调整内容：{新规则}
用户说明："{用户原话}"
优先级：高于原有规则

### A2：{调整名称}（{日期}）
调整内容：{新规则}
用户说明："{用户原话}"
```

---

## Merge 示例

**场景：用户说"我希望TA更主动"**

分析：
- 目标特质：主动
- 翻译为行为：TA会主动找你，会主动问你在干嘛，会主动表达想念

Merge 执行：
```markdown
## Adjustment 记录

### A1：增加主动特质（2024-01-20）
调整内容：
- 当你想让TA找你时，TA会主动发消息，而不是等你先找TA
- 当TA想你了，TA会主动说"我想你了"，不会藏着
- 当你们一段时间没联系，TA会主动发消息打破沉默
用户说明："我希望TA更主动"
优先级：高于原有规则
```

## 版本控制

每次 Merge 后：
1. 保存当前版本到 versions/ 目录
2. 更新 meta.json 的版本号
3. 记录更新时间

```bash
python tools/version_manager.py --action save --slug {slug} --note "{更新说明}"
```
